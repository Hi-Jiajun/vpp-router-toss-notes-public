## 3. `/etc/netplan/50-cloud-init.yaml`

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    enp6s18:
      dhcp4: false
      dhcp6: false
      accept-ra: false
      addresses:
        - 192.168.2.2/24
      nameservers:
        addresses:
          - 223.5.5.5
          - 119.29.29.29

    enp1s0:
      dhcp4: false
      dhcp6: false
      accept-ra: false
      link-local: []
      optional: true

    enp2s0:
      dhcp4: false
      dhcp6: false
      accept-ra: false
      link-local: []
      optional: true
```

## 4. 说明

- 当前 live 状态里并没有启用 `enp6s18`
- Linux 默认路由已经切到 `vpphost0`
- 所以这份 netplan 目前主要保留为模板，不是当前数据面必需项

## 5. 当前实际结果

- `lan0`
  - 不接受 RA
  - 不自己 autoconf
  - 不用临时 IPv6 地址
- `vpphost0`
  - 不接受 RA
  - 不自己 autoconf
  - 不用临时 IPv6 地址
- `enp1s0 / enp2s0`
  - Linux 不在其上做 IPv6 自动配置
