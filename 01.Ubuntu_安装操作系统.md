# Ubuntu 安装操作系统

本章记录当前这套路由虚拟机安装 `Ubuntu Server` 的方式。

整体安装界面和常规流程与下面这篇参考文档非常接近：

- [01.Ubuntu_安装操作系统.md](https://gitee.com/callmer/linux_router_toss_notes/blob/main/01.Ubuntu_%E5%AE%89%E8%A3%85%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F.md)

为了避免与原文过于接近，本文不重复展开所有“未修改”的标准安装界面，只记录当前环境下需要明确选择或与参考文档不同的部分。

## 1. 安装 Ubuntu Server

在上一篇文章 [00.PVE_创建路由虚拟机](./00.PVE_创建路由虚拟机.md) 中，已经完成路由虚拟机创建。

接下来：

- 挂载 `Ubuntu Server` 安装镜像
- 通过 `PVE Console` 启动虚拟机
- 进入安装器后按顺序完成系统安装

### 1.1 系统语言

当前建议保持：

- `English`

原因很简单：

- 服务器系统和命令行环境更容易与官方文档保持一致
- 后续软件日志、报错和配置项更容易检索

### 1.2 键盘布局

当前建议保持：

- `English (US)`

### 1.3 安装模式

这里是当前环境与参考文档的一个明确差异：

- 当前选择：`Ubuntu Server (minimized)`

这样做的原因是：

- 这台虚拟机只承担路由相关工作
- 尽量减少默认安装的软件包
- 后续需要的软件再按需补装

也正因为使用了 `minimized`，下一章会额外安装：

- `ping`
- `nslookup`

用于基础连通性和 DNS 调试。

### 1.4 网络接口

安装阶段建议只保留一个“临时能上网”的接口启用 DHCP，其余接口先关闭自动获取地址。

当前这台路由虚拟机有两块直通网卡，因此安装时尤其要注意：

- 只让一个接口承担安装期间的联网
- 其他接口避免在安装阶段误接入现网并获取地址

如果安装器里多个接口都拿到了地址，建议只保留用于临时联网的那个接口，其余接口把 `IPv4` 改为 `Disabled`。

### 1.5 网络代理

当前环境下：

- `Proxy address` 留空

### 1.6 系统镜像站

如果安装器默认镜像站可用，可以继续使用。

如果需要提高下载速度或改用国内源，可参考原始文档中的镜像站配置思路：

- [01.Ubuntu_安装操作系统.md](https://gitee.com/callmer/linux_router_toss_notes/blob/main/01.Ubuntu_%E5%AE%89%E8%A3%85%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F.md)

这里的镜像站设置主要只是为了顺利完成安装；系统装好后，软件源会在下一章再单独整理。

### 1.7 磁盘布局

当前虚拟机的系统盘是：

- `32G`
- `scsi0`

如果没有特殊需求，安装阶段保持“单系统盘”思路即可。

本文没有强绑定某一种分区方案；如果你当前安装方式与参考文档一致，可直接按参考文档对应小节操作。

### 1.8 系统账户

建议在安装阶段直接创建一个具有 `sudo` 权限的普通用户，并设置强密码。

这台机器后续会长期承担路由职责，因此：

- 用户名建议清晰易记
- 密码不要过弱
- 不建议依赖 `root` 直接密码登录

### 1.9 SSH 服务

当前建议：

- 勾选 `Install OpenSSH server`

这样系统安装完成后，可以尽快通过 `ssh` 进入机器进行后续配置。

### 1.10 附加组件

安装阶段不强制要求选择额外组件。

对于这台以路由为目的的 `minimized` 虚拟机，当前做法是：

- 安装阶段尽量少选
- 系统启动后再按需安装

## 2. 安装完成后处理

系统安装完成并重启后，建议先在控制台确认：

- 系统能正常进入登录界面
- 临时联网接口已经获取地址
- `OpenSSH` 可用

如果控制台没有直接显示地址，可以执行：

```bash
ip a
```

把当前可登录的管理地址记下来，后续配置会持续用到。
