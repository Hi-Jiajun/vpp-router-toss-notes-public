# vpp_router_toss_notes

VPP 家用主路由折腾笔记与复现文档。

可以把这页当作 `01. 总览与阅读顺序`。

这份仓库记录的是一套在真实家庭网络中跑通过的 `VPP + Linux` 主路由方案，重点是：

- `VPP` 负责主数据面
- `Linux` 只负责控制面和辅助服务

它更像一份“实战复现笔记”，而不是可直接照搬到所有环境的通用模板。

## 方案概览

当前这套方案的核心组件包括：

- `VPP` 负责主数据面
  - `PPPoE`
  - `NAT44 EI`
  - `IPv6 PD`
  - Telegram IPv4/IPv6 静态路由
- `Linux` 负责控制面
  - `dnsmasq`
  - `LCP` 对出的 `lan0`
  - `vpphost0` 作为 Linux 通过 VPP 上网的内部出口

## 前置假设

阅读和复现时，默认你已经具备这些基础条件：

- 一台运行 Linux 的路由主机
- 至少两个网络口，分别承担 `WAN` 和 `LAN`
- `systemd` 环境
- 上游为 `PPPoE`
- 下游是一段独立的家庭 LAN
