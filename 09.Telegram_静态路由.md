# Telegram 静态路由

## 1. IPv4 路由

文件：

- `/etc/vpp/routes.telegram.vpp`

内容：

```bash
ip route add 149.154.160.0/20 via 192.168.1.4
ip route add 185.76.151.0/24 via 192.168.1.4
ip route add 91.105.192.0/23 via 192.168.1.4
ip route add 91.108.12.0/22 via 192.168.1.4
ip route add 91.108.16.0/22 via 192.168.1.4
ip route add 91.108.20.0/22 via 192.168.1.4
ip route add 91.108.4.0/22 via 192.168.1.4
ip route add 91.108.56.0/22 via 192.168.1.4
ip route add 91.108.8.0/22 via 192.168.1.4
```

## 2. IPv6 路由

文件：

- `/etc/vpp/routes.telegram6.vpp`

内容：

```bash
ip route add 2001:67c:4e8::/48 via fd00:1234:5678:1::254 lan0phy
ip route add 2001:b28:f23c::/48 via fd00:1234:5678:1::254 lan0phy
ip route add 2001:b28:f23d::/48 via fd00:1234:5678:1::254 lan0phy
ip route add 2001:b28:f23f::/48 via fd00:1234:5678:1::254 lan0phy
ip route add 2a0a:f280::/32 via fd00:1234:5678:1::254 lan0phy
```

## 3. 自动更新脚本

### IPv4

文件：

- `/usr/local/sbin/update-telegram-routes.sh`

当前行为：

- 下载 Telegram 官方 IPv4 CIDR
- 更新 `/etc/vpp/routes.telegram.vpp`
- **当前仍会自动 `restart vpp`**

### IPv6

文件：

- `/usr/local/sbin/update-telegram-routes6.sh`

当前行为：

- 下载 Telegram 官方 IPv6 CIDR
- 更新 `/etc/vpp/routes.telegram6.vpp`
- 不自动重启 VPP

## 4. systemd timer

启用中的 timer：

- `update-telegram-routes.timer`
- `update-telegram-routes6.timer`

当前计划时间：

- IPv4：每天 `04:30`
- IPv6：每天 `04:40`

## 5. 验证命令

```bash
vppctl show ip fib | grep '192.168.1.4'
vppctl show ip6 fib | grep 'fd00:1234:5678:1::254'
```
