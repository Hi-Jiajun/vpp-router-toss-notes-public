# Ubuntu 安装系统软件

本章记录当前这套路由虚拟机在完成 `Ubuntu Server` 最小化安装后，补充系统软件的方式。

参考结构来源：

- [02.Ubuntu_安装系统软件.md](https://gitee.com/callmer/linux_router_toss_notes/blob/main/02.Ubuntu_%E5%AE%89%E8%A3%85%E7%B3%BB%E7%BB%9F%E8%BD%AF%E4%BB%B6.md)

但当前环境和参考文档有一个明显差异：

- 当前系统使用的是 `Ubuntu Server (minimized)`

因此这里的目标不是一次性装一大堆常驻工具，而是先补齐：

- 维护系统所需的最小工具
- `PVE` 虚拟机所需的 `qemu-guest-agent`
- 调试时最常用的 `ping` 和 `nslookup`

其他工具一律按需安装。

## 1. 配置软件源

如果你在安装系统阶段已经选用了可正常访问的镜像站，这一步可以只做检查，不一定要立即更换。

如果你需要改源，建议参考原始文档中的软件源配置方式：

- [02.Ubuntu_安装系统软件.md](https://gitee.com/callmer/linux_router_toss_notes/blob/main/02.Ubuntu_%E5%AE%89%E8%A3%85%E7%B3%BB%E7%BB%9F%E8%BD%AF%E4%BB%B6.md)

原因是：

- `Ubuntu 24.04` 已经采用 `DEB822` 风格的软件源配置
- 直接贴一整段镜像配置容易和参考文档过于接近

如果当前默认源可用，也完全可以先不改，直接进入更新和安装阶段。

## 2. 更新系统

建议先清理旧缓存，再同步软件源并完成系统升级：

```bash
# 清理已下载但不再需要的包缓存
sudo apt clean
# 进一步清理过期缓存
sudo apt autoclean
# 删除当前系统不再需要的依赖包
sudo apt autoremove --purge -y
# 同步软件源索引
sudo apt update
# 完整升级到当前源中的最新状态
sudo apt full-upgrade -y
```

如果这一步升级了较多基础组件，建议主动重启一次：

```bash
sudo reboot
```

## 3. 安装系统软件

### 3.1 先补齐最小化系统缺失的基础工具

由于当前安装模式是 `minimized`，建议至少补上：

```bash
# nano: 先补一个最基础的文本编辑器
# iputils-ping: 提供 ping 命令
# dnsutils: 提供 nslookup 等基础 DNS 调试工具
sudo apt install -y nano iputils-ping dnsutils
```

这里分别对应：

- `nano`
  - 至少要先有一个顺手的文本编辑器
- `iputils-ping`
  - 提供 `ping`
- `dnsutils`
  - 提供 `nslookup`

这三样是后续调试最常用、也最不该缺的基础工具。

### 3.2 安装 PVE 虚拟机需要的代理组件

因为当前系统运行在 `PVE` 虚拟机中，建议安装并启用：

```bash
# 安装 QEMU Guest Agent，方便 PVE 获取来宾 IP、做优雅关机
sudo apt install -y qemu-guest-agent
# 设置开机自启并立即启动
sudo systemctl enable --now qemu-guest-agent
```

安装后可以验证：

```bash
systemctl status qemu-guest-agent
```

### 3.3 按需补装维护工具

下面这些不强制，但很常用，可以按你的实际习惯补装：

```bash
# curl: HTTP/HTTPS 连通性测试
# pciutils: 提供 lspci
# ethtool: 查看网卡能力、驱动和链路状态
sudo apt install -y curl pciutils ethtool
```

分别用于：

- `curl`
  - 测试 HTTP/HTTPS 访问
- `pciutils`
  - 提供 `lspci`，方便确认直通网卡
- `ethtool`
  - 查看网卡能力和链路状态

如果你暂时不需要，也可以先不装，后续缺什么再补什么。

## 4. 当前最小推荐集

如果只保留当前这套路由虚拟机最小、够用的一组软件，建议至少安装：

```bash
# 最小可维护集合：编辑器、guest agent、ping、DNS 调试
sudo apt install -y nano qemu-guest-agent iputils-ping dnsutils
# 让 guest agent 立即生效并随系统启动
sudo systemctl enable --now qemu-guest-agent
```

## 5. 安装后核对

安装完成后，建议先确认下面这些命令可正常使用：

```bash
# 验证主机自身基础出网
ping -c 3 223.5.5.5
# 验证 DNS 解析是否正常
nslookup www.baidu.com
# 验证 guest agent 是否已经启动
systemctl status qemu-guest-agent
```

如果还需要进一步确认直通设备是否已经在系统中可见，也可以执行：

```bash
# 确认两块直通网卡是否已在客户机内部可见
lspci -nn
```
