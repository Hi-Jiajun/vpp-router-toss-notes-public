# 内网辅助节点说明

本章用于说明当前这套路由环境里几个关键内网地址分别对应什么项目，以及它们和主路由之间的关系。

这些地址虽然不直接承担主路由数据面，但会影响：

- 静态路由的下一跳
- 上游 DNS 解析链路
- fakeip / Telegram 分流

## 1. `192.168.1.4` / `fd00:1234:5678:1::254`

当前：

- `192.168.1.4`
- `fd00:1234:5678:1::254`

实际上是同一台内网辅助设备的：

- `IPv4` 地址
- `IPv6` 地址

这台设备对应的项目是：

- [kkkgo/PaoPaoGateWay](https://github.com/kkkgo/PaoPaoGateWay)

根据该项目 README，它本身定位为一个基于 Linux 的 `FakeIP` 网关方案。

在当前这套路由配置里，这台设备主要承担：

- `7.0.0.0/8` fakeip 流量的特殊下一跳
- Telegram IPv4 静态路由的下一跳
- Telegram IPv6 静态路由的下一跳

也就是说：

- `192.168.1.4` 是它的 `IPv4` 下一跳地址
- `fd00:1234:5678:1::254` 是同一台设备的 `IPv6` 下一跳地址

这里特别要注意：

- 它不是整个 LAN 的默认网关
- 它只承担特定业务流量的旁路下一跳
- 它不应该在客户端侧被当成整段 LAN 的默认 IPv6 router

## 2. `192.168.1.2`

当前：

- `192.168.1.2`

对应的项目是：

- [kkkgo/PaoPaoDNS](https://github.com/kkkgo/PaoPaoDNS)

根据该项目 README，它是一个面向递归解析场景的 DNS 服务组合方案。

在当前这套路由配置里，它的角色是：

- 作为本机 `mini-ppdns` 的主上游 DNS
- 为 LAN 侧客户端提供更稳定的递归解析能力

对应到当前 `mini-ppdns` / DNS 链路里，可理解为主上游：

```text
[dns]
192.168.1.2:53
```

其中：

- `192.168.1.2` 是它的 `IPv4` 地址
- `fd00:1234:5678:1::54` 是它当前使用的 `IPv6` 地址

## 3. `192.168.1.250`

当前：

- `192.168.1.250`

对应的项目是：

- [kkkgo/mini-ppdns](https://github.com/kkkgo/mini-ppdns)

根据该项目 README，它是一个轻量级的 DNS 故障转移转发器。

在当前这套路由配置里，它更适合被理解为：

- 一个部署了 `mini-ppdns` 项目的内网辅助节点
- 当前文档中仍保留它的地址说明和历史角色信息
- 但根据主路由最新 live 配置，它已经不是主路由 `dnsmasq` 的直接上游

也就是说，当前主路由真正新增的是：

- 本机 `192.168.1.1:53` 由本机 `mini-ppdns` 承担

而不是继续让 `dnsmasq` 直接把请求转发给 `192.168.1.250`。

它对应的地址信息仍然如下：

```text
192.168.1.250
fd00:1234:5678:1::53
```

其中：

- `192.168.1.250` 是它的 `IPv4` 地址
- `fd00:1234:5678:1::53` 是它当前使用的 `IPv6` 地址

## 4. 主路由本机新增的 mini-ppdns

根据当前主路由 live 配置，本机已经新增部署：

- `/usr/sbin/mini-ppdns`
- `/usr/sbin/mini-ppdns.sh`
- `/etc/mini-ppdns.ini`

当前 DNS 解析链路更准确的描述是：

- LAN Client -> `192.168.1.1:53`
- `192.168.1.1:53` -> 主路由本机 `mini-ppdns`
- `mini-ppdns` -> `192.168.1.2` 主上游 / 公共 fallback

## 5. 当前关系总览

如果把这几个辅助节点和主路由之间的关系压缩成一句话，大致就是：

- 主路由的数据面由 `VPP` 负责
- `192.168.1.4` 这台 `PaoPaoGateWay` 负责特定静态路由流量的旁路下一跳
- 主路由本机的 `mini-ppdns` 现在负责 `192.168.1.1:53`
- `192.168.1.2` 的 `PaoPaoDNS` 是本机 `mini-ppdns` 的主上游 DNS
