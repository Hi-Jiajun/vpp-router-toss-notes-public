# 内网辅助节点说明

本章用于说明当前这套路由环境里几个关键内网地址分别对应什么项目，以及它们和主路由之间的关系。

这些地址虽然不直接承担主路由数据面，但会影响：

- 静态路由的下一跳
- 上游 DNS 解析链路
- fakeip / Telegram 分流

## 1. `192.168.1.4` / `fd00:1234:5678:1::254`

当前：

- `192.168.1.4`
- `fd00:1234:5678:1::254`

实际上是同一台内网辅助设备的：

- `IPv4` 地址
- `IPv6` 地址

这台设备对应的项目是：

- [kkkgo/PaoPaoGateWay](https://github.com/kkkgo/PaoPaoGateWay)

根据该项目 README，它本身定位为一个基于 Linux 的 `FakeIP` 网关方案。

在当前这套路由配置里，这台设备主要承担：

- `7.0.0.0/8` fakeip 流量的特殊下一跳
- Telegram IPv4 静态路由的下一跳
- Telegram IPv6 静态路由的下一跳

也就是说：

- `192.168.1.4` 是它的 `IPv4` 下一跳地址
- `fd00:1234:5678:1::254` 是同一台设备的 `IPv6` 下一跳地址

这里特别要注意：

- 它不是整个 LAN 的默认网关
- 它只承担特定业务流量的旁路下一跳
- 它不应该在客户端侧被当成整段 LAN 的默认 IPv6 router

## 2. `192.168.1.2`

当前：

- `192.168.1.2`

对应的项目是：

- [kkkgo/PaoPaoDNS](https://github.com/kkkgo/PaoPaoDNS)

根据该项目 README，它是一个面向递归解析场景的 DNS 服务组合方案。

在当前这套路由配置里，它的角色是：

- 作为 `dnsmasq` 的一个上游 DNS 服务器
- 为 LAN 侧客户端提供更稳定的递归解析能力

对应到当前 `dnsmasq` 配置里，就是这些上游项之一：

```text
server=192.168.1.2
server=fd00:1234:5678:1::54
```

其中：

- `192.168.1.2` 是它的 `IPv4` 地址
- `fd00:1234:5678:1::54` 是它当前使用的 `IPv6` 地址

## 3. `192.168.1.250`

当前：

- `192.168.1.250`

对应的项目是：

- [kkkgo/mini-ppdns](https://github.com/kkkgo/mini-ppdns)

根据该项目 README，它是一个轻量级的 DNS 故障转移转发器。

在当前这套路由配置里，它的角色是：

- 作为 `dnsmasq` 的另一个上游 DNS 服务器
- 为解析链路提供额外的容灾和兜底能力

对应到当前 `dnsmasq` 配置里，就是这些上游项之一：

```text
server=192.168.1.250
server=fd00:1234:5678:1::53
```

其中：

- `192.168.1.250` 是它的 `IPv4` 地址
- `fd00:1234:5678:1::53` 是它当前使用的 `IPv6` 地址

## 4. 当前关系总览

如果把这几个辅助节点和主路由之间的关系压缩成一句话，大致就是：

- 主路由的数据面由 `VPP` 负责
- `192.168.1.4` 这台 `PaoPaoGateWay` 负责特定静态路由流量的旁路下一跳
- `192.168.1.2` 的 `PaoPaoDNS` 和 `192.168.1.250` 的 `mini-ppdns` 共同作为 `dnsmasq` 的上游 DNS
