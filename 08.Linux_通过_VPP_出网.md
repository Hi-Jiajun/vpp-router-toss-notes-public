# Linux 通过 VPP 出网

## 1. 目标

这部分的目标不是让 Linux 自己承担路由功能，而是让 Linux：

- `apt update`
- `curl`
- `git clone`

这类维护动作能通过 VPP 的 NAT 出口正常联网。

## 2. VPP 配置

文件：

- `/etc/vpp/linux-egress.vpp`

内容：

```bash
# 在 VPP 侧创建 tap，并同时在 Linux 侧生成 vpphost0
create tap id 100 if-name hosttap0 host-if-name vpphost0 host-ip4-addr 10.255.255.2/30 host-ip4-gw 10.255.255.1
# 拉起 VPP 侧 tap 接口
set interface state hosttap0 up
# 给 VPP 侧接口配置 /30 地址
set interface ip address hosttap0 10.255.255.1/30
# 把 Linux 主机自己的流量也纳入 NAT44 EI
set interface nat44 ei in hosttap0 out pppox0
```

## 3. Linux live 状态

当前 Linux 侧：

- `vpphost0 = 10.255.255.2/30`
- 默认路由：

```text
default via 10.255.255.1 dev vpphost0
```

## 4. 设计说明

- `hosttap0` 是 VPP 侧接口
- `vpphost0` 是 Linux 侧接口
- Linux 只走 IPv4 通过该口出网
- IPv6 不通过 `vpphost0` 做 Linux 自身上网

## 5. 验证命令

```bash
# 查看 Linux 侧控制面接口地址
ip -br a show vpphost0
# 查看 Linux 默认路由是否已经切到 vpphost0
ip route
# 测试到 VPP 侧 /30 对端是否可达
ping -c 3 10.255.255.1
# 测试 Linux 主机自身能否借道出网
ping -c 3 223.5.5.5
# 再做一次 HTTP 层验证
curl -4 -I --max-time 8 https://www.baidu.com
```
