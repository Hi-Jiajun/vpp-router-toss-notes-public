# Ubuntu 设置系统参数

本章记录当前这套路由虚拟机安装完系统软件后，需要额外调整的系统参数。

参考结构来源：

- [03.Ubuntu_设置系统参数.md](https://gitee.com/callmer/linux_router_toss_notes/blob/main/03.Ubuntu_%E8%AE%BE%E7%BD%AE%E7%B3%BB%E7%BB%9F%E5%8F%82%E6%95%B0.md)

但当前环境并不是一台通用 Linux Router，而是：

- 运行在 `PVE` 中的路由虚拟机
- `VPP` 负责主数据面
- Linux 只负责控制面和辅助服务

所以这一步的重点不是把 Linux 内核调成“主路由转发器”，而是：

- 关闭不需要的自动网络行为
- 固化 `IPv4 / IPv6` 相关安全和接口参数
- 保留一份可复现的 `netplan` 基线模板

## 1. 设置 sysctl 基线参数

### 1.1 路由器基础参数

文件：

- `/etc/sysctl.d/90-router-base.conf`

内容：

```conf
# Linux 不承担主数据面转发，保持内核转发关闭
net.ipv4.ip_forward=0
net.ipv6.conf.all.forwarding=0

# 关闭 rp_filter，避免非对称或旁路路径被内核误判
net.ipv4.conf.all.rp_filter=0
net.ipv4.conf.default.rp_filter=0

# 不接受和发送 IPv4 redirect，也不接受 source route
net.ipv4.conf.all.accept_redirects=0
net.ipv4.conf.default.accept_redirects=0
net.ipv4.conf.all.send_redirects=0
net.ipv4.conf.default.send_redirects=0
net.ipv4.conf.all.accept_source_route=0
net.ipv4.conf.default.accept_source_route=0

# 关闭 IPv6 redirect 与 source route
net.ipv6.conf.all.accept_redirects=0
net.ipv6.conf.default.accept_redirects=0
net.ipv6.conf.all.accept_source_route=0
net.ipv6.conf.default.accept_source_route=0

# 关闭 RA / autoconf / 临时地址，避免 Linux 自己抢着当网络控制者
net.ipv6.conf.all.accept_ra=0
net.ipv6.conf.default.accept_ra=0
net.ipv6.conf.all.autoconf=0
net.ipv6.conf.default.autoconf=0
net.ipv6.conf.all.use_tempaddr=0
net.ipv6.conf.default.use_tempaddr=0

# 给控制面保留相对稳妥的网络缓冲区参数
net.core.rmem_max=33554432
net.core.wmem_max=33554432
net.core.netdev_max_backlog=4096

# 降低 swap 积极性，减少常驻路由虚拟机的抖动
vm.swappiness=10
```

这份基础参数的核心思路是：

- Linux 内核不承担真正的数据面转发
- 关闭重定向、源路由、RA 自动配置等不需要的行为
- 保留较稳妥的网络缓冲区参数

这里特别要注意：

- `net.ipv4.ip_forward=0`
- `net.ipv6.conf.all.forwarding=0`

这是符合当前方案设计的。

因为：

- 真正负责转发的是 `VPP`
- Linux 本身只通过 `vpphost0` 借道出网

### 1.2 接口级 IPv6 参数

文件：

- `/etc/sysctl.d/91-router-ipv6-lan.conf`

内容：

```conf
# 关闭 lan0 上的 IPv4 redirect / source route
net.ipv4.conf.lan0.accept_redirects=0
net.ipv4.conf.lan0.send_redirects=0
net.ipv4.conf.lan0.accept_source_route=0

# 关闭 vpphost0 上的 IPv4 redirect / source route
net.ipv4.conf.vpphost0.accept_redirects=0
net.ipv4.conf.vpphost0.send_redirects=0
net.ipv4.conf.vpphost0.accept_source_route=0

# lan0 只作为 Linux 感知 LAN 的 host 接口，不自己接 RA / autoconf
net.ipv6.conf.lan0.accept_ra=0
net.ipv6.conf.lan0.autoconf=0
net.ipv6.conf.lan0.use_tempaddr=0
net.ipv6.conf.lan0.accept_redirects=0

# vpphost0 只给 Linux 自身出网，不让它参与 IPv6 自动配置
net.ipv6.conf.vpphost0.accept_ra=0
net.ipv6.conf.vpphost0.autoconf=0
net.ipv6.conf.vpphost0.use_tempaddr=0
net.ipv6.conf.vpphost0.accept_redirects=0

# 两块直通物理网卡最终交给 VPP，用 Linux 视角先保持安静
net.ipv6.conf.enp1s0.accept_ra=0
net.ipv6.conf.enp1s0.autoconf=0
net.ipv6.conf.enp1s0.use_tempaddr=0
net.ipv6.conf.enp1s0.accept_redirects=0

net.ipv6.conf.enp2s0.accept_ra=0
net.ipv6.conf.enp2s0.autoconf=0
net.ipv6.conf.enp2s0.use_tempaddr=0
net.ipv6.conf.enp2s0.accept_redirects=0
```

这里主要约束的是：

- `lan0`
- `vpphost0`
- 两块物理直通网卡

目标是避免 Linux 自己在这些接口上：

- 接收 RA
- 自动生成 IPv6 地址
- 参与不需要的 IPv6 默认路由选择

## 2. 设置 netplan 基线模板

当前环境中，`netplan` 不是数据面核心，但仍然保留了一份基线模板，方便后续维护和恢复。

文件：

- `/etc/netplan/50-cloud-init.yaml`

内容：

```yaml
network:
  version: 2
  renderer: networkd
  ethernets:
    # 预留的虚拟管理网卡模板：当前 live 状态不作为主出口
    enp6s18:
      dhcp4: false
      dhcp6: false
      accept-ra: false
      addresses:
        - 192.168.2.2/24
      nameservers:
        addresses:
          - 223.5.5.5
          - 119.29.29.29

    # 直通物理网卡 1：留给 VPP 使用，Linux 自己不做自动配置
    enp1s0:
      dhcp4: false
      dhcp6: false
      accept-ra: false
      link-local: []
      optional: true

    # 直通物理网卡 2：留给 VPP 使用，Linux 自己不做自动配置
    enp2s0:
      dhcp4: false
      dhcp6: false
      accept-ra: false
      link-local: []
      optional: true
```

## 3. 当前设计说明

这份 `netplan` 和 `sysctl` 组合的设计目的，不是让 Linux 自己把所有网口管起来，而是让 Linux 维持一个“尽量安静”的控制面环境。

当前需要理解这几点：

1. live 状态里并没有真正启用 `enp6s18`(虚拟网卡/管理网卡) 作为当前主出口。
2. Linux 默认路由已经切到 `vpphost0`。
3. `enp1s0 / enp2s0` 只是客户机内看到的直通网卡，不应该在 Linux 自己这里做 IPv6 自动配置。
4. 这份 `netplan` 更多是模板和兜底基线，不是当前数据面成立的关键条件。

## 4. 应用参数

修改完成后，可以执行：

```bash
# 重新加载全部 sysctl 配置
sudo sysctl --system
# 只做 netplan 语法与渲染检查，不立即切换 live 状态
sudo netplan generate
```

如果确实需要把 `netplan` 配置下发到 live 系统，再执行：

```bash
# 将 netplan 真正应用到 live 网络环境，操作前要确认不会断管理面
sudo netplan apply
```

但当前这套路由方案里，`netplan apply` 应该谨慎进行，确认不会影响现有管理面连接后再操作。

## 5. 当前实际结果

当前希望看到的 live 结果大致是：

- `lan0`
  - 不接受 RA
  - 不自己 `autoconf`
  - 不使用临时 IPv6 地址
- `vpphost0`
  - 不接受 RA
  - 不自己 `autoconf`
  - 不使用临时 IPv6 地址
- `enp1s0 / enp2s0`
  - Linux 不在其上做 IPv6 自动配置
