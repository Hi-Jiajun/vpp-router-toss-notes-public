# dnsmasq 与 mini-ppdns

本章记录当前这套路由方案里，Linux 控制面上的 DHCP / RA / DNS 是如何分工的。

当前实际分工已经和最初版本不同：

- `dnsmasq`
  - 负责 `DHCPv4`
  - 负责 `RA`
  - 负责 `IPv6 SLAAC / DHCPv6` 地址下发
  - 不再直接监听 `53` 提供 DNS 服务
- `mini-ppdns`
  - 负责主路由本机上的 DNS 服务
  - 监听 `0.0.0.0:53`
  - 上游主 DNS 为 `192.168.1.2`
  - 不可用时自动切到公共 fallback DNS

## 1. 当前职责划分

当前 LAN 客户端拿到的 DNS 仍然是：

- `192.168.1.1`

但这里需要特别注意：

- 现在真正对 `192.168.1.1:53` 响应的是本机 `mini-ppdns`
- 不再是 `dnsmasq`

也就是说，当前主路由上的 DNS 链路大致变成：

```text
LAN Client -> 192.168.1.1:53 (mini-ppdns) -> 192.168.1.2:53 (main) / public fallback
```

而 `dnsmasq` 则退回到：

- DHCP
- RA
- 地址下发
- 静态地址绑定

## 2. dnsmasq 主配置

文件：

- `/etc/dnsmasq.d/router.conf`

当前实际内容如下：

```conf
# 把 dnsmasq 日志写到单独文件，便于排障
log-facility=/var/log/dnsmasq.log
log-async=20

# 关键变化：关闭 DNS 监听功能，只保留 DHCP / RA 职责
port=0

# 只监听 lan0，避免误在其他接口提供服务
interface=lan0
bind-dynamic
listen-address=192.168.1.1

# 基础安全与行为约束
domain-needed
bogus-priv
no-resolv
strict-order
no-round-robin
cache-size=0
local-service
rebind-localhost-ok
stop-dns-rebind
dhcp-authoritative
log-dhcp

# 这些和 DNS 直接相关的本地域名 / host-record 当前被注释掉
# 因为主路由上的 DNS 解析已改由 mini-ppdns 承接
#domain=example.internal
#local=/example.internal/
#expand-hosts
#host-record=router.example.internal,192.168.1.1,fd00:1234:5678:1::1
#host-record=mini-ppdns.example.internal,192.168.1.250,fd64:2580:8165:1:82ac:caff:feb5:5303

# 这些上游 DNS 配置也已注释掉
# 当前不再由 dnsmasq 直接负责 DNS 转发
#server=192.168.1.250
#server=192.168.1.2
#server=fd00:1234:5678:1::53
#server=fd00:1234:5678:1::54

# IPv4 DHCP
dhcp-range=192.168.1.100,192.168.1.200,255.255.255.0,12h
dhcp-option=option:router,192.168.1.1
# 客户端继续拿 192.168.1.1 做 DNS，但这里实际指向本机 mini-ppdns
dhcp-option=option:dns-server,192.168.1.1
# 本地域名搜索项当前也已注释
#dhcp-option=option:domain-search,example.internal
dhcp-option=option:classless-static-route,7.0.0.0/8,192.168.1.4
dhcp-option=tag:aliyun_google_dns,option:dns-server,223.5.5.5,8.8.8.8

# IPv6: stateful + SLAAC, but no IPv6 DNS
enable-ra
ra-param=lan0,mtu:1500,30,180
dhcp-range=::100,::1ff,constructor:lan0,slaac,64,12h
dhcp-range=fd00:1234:5678:1::100,fd00:1234:5678:1::1ff,slaac,64,12h
# 当前也没有显式下发 IPv6 DNS
#dhcp-option=option6:dns-server

dhcp-broadcast=tag:needs-broadcast
dhcp-ignore-names=tag:dhcp_bogus_hostname
dhcp-leasefile=/var/lib/misc/dnsmasq.leases
```

## 3. dnsmasq 静态地址

文件：

- `/etc/dnsmasq.d/dnsmasq.staticv4.conf`

当前实际内容如下：

```conf
# 这台 DNS 节点会额外拿到公共 DNS 作为补充
dhcp-host=02:00:00:00:00:16,set:aliyun_google_dns,192.168.1.2,dns-global,2h
# 这台地址当前保留给 mini-ppdns 项目相关节点
dhcp-host=02:00:00:00:00:18,192.168.1.250,ap-upstream,2h
```

说明：

- 当前 `dnsmasq.staticv4.conf` 中的有效记录已经比之前少很多
- 文档应以这份 live 配置为准，而不是旧版样例

## 4. 本机 mini-ppdns

当前主路由本机已经新增部署：

- `/usr/sbin/mini-ppdns`
- `/usr/sbin/mini-ppdns.sh`
- `/etc/mini-ppdns.ini`

### 4.1 启动脚本

文件：

- `/usr/sbin/mini-ppdns.sh`

当前脚本的职责大致是：

- 检查 `mini-ppdns` 是否正在运行
- 如果未运行则启动
- 写入 PID 文件
- 支持 `start / stop / restart`

这意味着它更像一个轻量 watchdog 脚本，而不是正式的 systemd service。

### 4.2 配置文件

文件：

- `/etc/mini-ppdns.ini`

当前实际配置如下：

```ini
[dns]
192.168.1.2:53

[fall]
61.139.2.69:53
218.6.200.139:53
223.5.5.5:53
8.8.8.8:53

[listen]
0.0.0.0:53

[force_fall]
192.168.1.2

[adv]
qtime=250
aaaa=noerror
lite=yes

[hook]
exec="curl -o /dev/null -s -w %{http_code} --proxy socks5h://192.168.1.4:1080 http://www.google.com/generate_204"
exit_code=0
keyword="204"
sleep_time=60
retry_mode=5
count=10
switch_fall_exec="curl -sk -o /dev/null --data 'Main DNS is DOWN!' --retry 3 https://ntfy.sh/mydns_status"
switch_main_exec="curl -sk -o /dev/null --data 'Main DNS is UP!' --retry 3 https://ntfy.sh/mydns_status"
```

这里可以这样理解：

- `[dns]`
  - 主上游 DNS 是 `192.168.1.2:53`，也就是 `PaoPaoDNS`
- `[fall]`
  - 当主上游不可用时，切到一组公共 fallback DNS
- `[listen]`
  - 本机监听 `0.0.0.0:53`
- `[force_fall]`
  - 对 `192.168.1.2` 做特殊处理
- `[hook]`
  - 通过 `192.168.1.4:1080` 的 `socks5h` 代理去访问 `generate_204`
  - 以 HTTP `204` 作为主 DNS 可用性的判断信号
  - 状态切换时通过 `ntfy.sh` 发送通知

## 5. 定时保活

当前 root 的 `crontab` 已增加：

```cron
* * * * * /usr/sbin/mini-ppdns.sh
```

这表示：

- 每分钟执行一次 `mini-ppdns.sh`
- 如果 `mini-ppdns` 没跑起来，就由脚本重新拉起

## 6. 当前设计说明

基于目前 live 配置，当前 DNS 职责已经变成：

1. `dnsmasq`
   - 负责 DHCP / RA / 地址下发
2. `mini-ppdns`
   - 负责主路由上的 DNS 服务
3. `PaoPaoDNS (192.168.1.2)`
   - 作为主上游 DNS
4. 公共 DNS
   - 作为 fallback

因此，`dnsmasq.d` 里那几段被注释掉的 DNS 相关配置，不是无意删掉，而是有意把主路由 DNS 职责迁移给 `mini-ppdns`。
